# 长期协作要求

本文件用于记录本项目后续希望 Codex 持续遵守的研究习惯。

## 基本原则

- 每次新增或修改代码后，在 `docs/change_log.md` 记录改动、用意、用法和验证情况。
- 每次实验输出放到 `outputs/` 下，按任务和日期/病例命名。
- 原始数据不入 git；`docs/` 和 `outputs/` 默认只在本地保存。
- 阶段一重点是 CT 中冈上肌腱候选 ROI 自动定位，不把任务偷换成精细分割。
- 第一阶段评价优先看 ROI recall、coverage、bbox IoU、候选体积占比和搜索空间缩小倍率。

## 实验记录模板

```text
日期：
目标：
输入数据：
命令：
关键参数：
结果：
失败/异常：
下一步：
```

## 结果汇总偏好

- 先给结论，再列证据。
- 对失败案例要写清楚可能原因：数据读取、方向/spacing、骨阈值、ROI 规则、标注不一致。
- 每次建议下一步时，尽量给出一个可以直接运行的最小实验。
